# 웹 소켓 방식이 왜 좋은가
1. 폴링: 주기적으로 데이터 요청
2. Server-Sent Event (SSE): 서버가 변경점이 있을 때마다 클라이언트한테 메세지 보냄. (단방향 통신)
3. 웹 소켓: TCP연결을 통해 양방향 통신 채널을 제공하는 기술. 걍 실시간 통신 ㅇㅇ

위에 폴링과 SSE는 양방향 통신에 부적합하다. 웹 소켓이 훌륭한 솔루션!
